# AHP 移动端交互设计方案

## 设计目标
为AHP(层次分析法)在移动端提供直观、高效的比较交互体验。

## 核心交互方案

### 连续滑动选择
- **单次滑动完成选择和程度设定**
- **向右滑动**：上方准则更重要 (1-9)
- **向左滑动**：下方准则更重要 (1-9)
- **中间位置**：同等重要 (1)

### 滑动映射逻辑
```
滑动位置 → 重要性值
中间(0)   → 1 (同等)
向右滑动 → 1.1, 1.2, 1.5, 2, 3, 5, 7, 9
向左滑动 → 0.9, 0.8, 0.5, 0.33, 0.2, 0.14, 0.11
```

## 视觉设计方案

### 1. 双色滑动条
```
[🔵───────●───────🟢]
价格更重要        质量更重要
```

**实现细节：**
- 滑动条左侧：准则A的颜色 (蓝色)
- 滑动条右侧：准则B的颜色 (绿色)
- 滑块位置：显示当前选择
- 中间点：同等重要

### 2. 大小缩放反馈
- **更重要准则**：放大显示 (1.0x → 1.2x)
- **较不重要准则**：缩小显示 (1.0x → 0.8x)

### 3. 颜色区分
- **准则A**：蓝色系
- **准则B**：绿色系
- **同等重要时**：保持各自颜色，大小相同

## 完整界面设计

```
价格 vs 质量
┌─────────────────┐
│   🔵 价格       │ ← 大小根据重要性调整
│   (描述)        │
├─────────────────┤
│   [🔵──●──🟢]    │ ← 双色滑动条
│   价格比质量稍强(3)│
├─────────────────┤
│   🟢 质量       │ ← 大小根据重要性调整
│   (描述)        │
└─────────────────┘
```

## 交互流程

1. **进入比较页面**：显示两个准则和双色滑动条
2. **滑动选择**：用户滑动选择重要性和程度
3. **实时反馈**：
   - 滑动条颜色和位置更新
   - 准则大小实时调整
   - 重要性文本描述更新
4. **确认选择**：点击继续进入下一组比较

## 技术实现要点

### 组件结构
- Material-UI Slider 组件
- CSS linear-gradient 实现双色滑动条
- CSS transform: scale() 实现大小缩放
- React state 管理滑动位置和重要性值

### 状态管理
```typescript
interface AHPComparisonState {
  currentValue: number; // 1-9 或 1/9-1/1
  criterion1: Criterion;
  criterion2: Criterion;
  progress: number;
}
```

### 响应式设计
- 大触摸区域确保移动端友好
- 垂直布局适合单手操作
- 适应不同屏幕尺寸
